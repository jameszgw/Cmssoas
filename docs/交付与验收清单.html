六、发版操作说明（Release）

CI 已配置：推送 v* 标签触发 release 流水线 → 下载各任务制品（覆盖率 / E2E 报告 / 混淆制品）→ 打包 codeman-artifacts.zip + 生成发布说明 → 创建 GitHub Release 并附件。

发版步骤（任选其一）：

  1. 命令行打 tag 并推送（具备 tag 推送权限时）：
   git checkout main && git pull
   git tag -a v1.0.0 -m "CODEMAN v1.0.0"
   git push origin v1.0.0          # 触发 release 流水线
  1. GitHub 网页：Releases → Draft a new release → 选择/新建 tag v1.0.0 → Publish（同样触发流水线）。

版本号约定：遵循 SemVer（vMAJOR.MINOR.PATCH）。首个交付版本建议 v1.0.0。 前置条件：main 分支 CI 全绿（backend/sdk/sign-smoke/harden/frontend/e2e 均通过）。 回滚：删除错误 tag git push origin :refs/tags/vX.Y.Z 并在 GitHub 删除对应 Release。

注：受限的托管环境仅允许推送指定特性分支，refs/tags/* 推送会被拒（HTTP 403）； 发版需在具备标签推送权限的环境，或经合并入 main 后由仓库侧执行。
