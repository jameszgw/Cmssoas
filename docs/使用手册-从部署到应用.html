CODEMAN 使用手册 —— 从部署到应用(完整版)

软件授权运营平台:代码保护 + License 认证 + 多租户运营。本手册覆盖环境准备 → 部署 → 初始化 → 各功能操作 → SDK 接入 → 运维全流程。 配套:部署目录 deploy/ · 功能文档 docs/ · 会话交接 HANDOFF。初始账号 admin / 8888(首登强制改密)。

1. 产品概览与架构

  • 三大子系统:代码保护(混淆+加密+License 绑定)、License 认证(Ed25519/国密 SM2、离线+在线)、多租户运营平台(Vue3 控制台 + Spring Boot 后端)。
  • 组成:server/license-platform(后端)、web/console(运营控制台)、sdk/license-sdk(客户端验签)、examples/protected-app(代码保护示例)。
  • 端到端:运营方在控制台开通租户/签发 License → 客户集成 SDK 验签与门禁 → 在线通道心跳/吊销 → 计费/合同/发票闭环 → 客户自助门户查看。
  • 技术栈:Spring Boot 3.5 / Java 21 / Flyway / JPA;Vue3 + TS + Element Plus + Vite;PostgreSQL·MySQL·H2 + Redis;Prometheus + Grafana。

2. 环境要求

项要求
JDK17+(推荐 21);不支持 Java 8
数据库H2(开发) / PostgreSQL 16(生产推荐) / MySQL 5.7·8.0(库须 utf8mb4)
缓存Redis 7(多实例 nonce 防重放;单机可不启用)
构建Maven 3.9+;Node 20+(前端)
容器Docker + Docker Compose(可选,推荐)

3. 部署

3.1 Docker Compose(推荐,一键)

cp .env.example .env          # 试用;生产用 cp .env.prod.example .env 并改密钥/SMTP/数据库
docker compose up -d --build
# 控制台 http://localhost   后端 http://localhost:8080   账号 admin/8888
# 监控 Prometheus :9090 / Grafana :3000(admin/admin)
  • 用 MySQL:docker compose -f docker-compose.yml -f docker-compose.mysql.yml up -d mysql redis backend console
  • 启用本地大模型(智能客服):docker compose --profile ai up -d → docker compose exec ollama ollama pull qwen2.5:7b-instruct → .env 设 AI_ENABLED=true 重启 backend。

3.2 裸机(CentOS systemd + nginx)

cp deploy/codeman-backend.env.example /etc/codeman/backend.env   # 改数据库/密钥
sudo bash deploy/install.sh                                      # 装 systemd 服务 + nginx 反代
systemctl status codeman-backend

详见 docs/部署-CentOS7.md。

3.3 本地开发(无需 Docker)

cd server/license-platform && mvn spring-boot:run      # :8080(默认 H2,开箱即跑)
cd web/console && npm install && npm run dev           # :5173(代理到 :8080)

4. 数据库选择

  • H2:开发默认,内存库,重启即清空。
  • PostgreSQL:生产推荐,SPRING_PROFILES_ACTIVE=prod + DB_URL/DB_USER/DB_PASSWORD。
  • MySQL 5.7/8.0:SPRING_PROFILES_ACTIVE=mysql,库须 utf8mb4;连接串 characterEncoding=UTF-8(非 utf8mb4)。
  • 建表两法:① 应用启动 Flyway 自动建表 + 写系统种子(推荐);② 纯 SQL:执行 deploy/sql/schema-<dialect>.sql → init-admin.sql(可选)→ test-data.sql(可选),并设 SPRING_FLYWAY_ENABLED=false。
  • 详见 deploy/sql/README.md 与 DEPLOY.md#数据库支持。

5. 首次登录与系统初始化

  1. 打开控制台 → 用 admin / 8888 登录 → 按提示修改初始密码。
  2. 若存在「强制确认」须知,登录后弹层,勾选「我已阅读并同意」方可继续。
  3. 应用首启会自动:建表、写权限/角色/套餐种子、创建超管 admin 并授予全部权限。
  4. 建议:在「角色权限」核对权限,在「用户管理」为成员建号。

6. 角色权限与用户管理

  • 角色权限:权限树支持「无 / 只读 / 编辑 / 完全」多态;新增权限自动授予超管;VIEWER 默认只读。
  • 用户管理:新建用户指定角色,初始密码默认 Codeman@123(可自定义);重置密码 / 启停;新用户首登强制改密。

7. 业务操作(逐模块)

每页内置 💡 操作指引与浅色功能简介;业务数据用等宽字体、与说明文案区分。

7.1 租户管理

开通租户(名称/管理员邮箱/套餐)→ 系统建租户独立 Schema 并发开通邮件(发件箱异步)→ 管理员凭邮件激活、设密码、(可选)绑定 MFA。支持续期、查看隔离配置。

7.2 License 授权(全生命周期)

  • 签发:选产品/版型/模块/有效期/并发 → 服务端私钥签名生成 .lic(Ed25519 默认 / 国密 SM2)。
  • 续期 / 变更 / 吊销:操作即重签并记版本历史(diff 可查)。
  • 下载 .lic;导出 CSV。
  • 吊销名单(CRL):查看已签名 CRL 与公开分发地址 /pub/crl、公钥集 /pub/license/public-keys(离线客户端按 kid 验签后拒绝已吊销)。
  • 到期停用:手动触发或每日自动将过期 License 置 EXPIRED。

7.3 在线监控

查看 SDK 在线实例、心跳、席位占用、断网宽限期与席位释放。

7.4 产品与版本 / 套餐订阅

维护产品矩阵与套餐(价格/席位/模块/功能点/版本范围);创建订阅 → 自动出账(账单 PENDING),可联动签发。

7.5 客户管理(客户360)

维护客户主数据;点「客户360」按客户名称聚合其 License/合同/账单/支付/订阅与关键指标。

7.6 计费账单(收款 + 开票)

  • 订阅/合同自动出账。
  • 收款:发起扫码收款 → 客户支付后财务核对到账,点「确认到账(人工)」完成(默认人工确认;可接微信/支付宝/Stripe,改 app.pay.provider)。
  • 开票:对已收款账单开具电子发票(普票/专票、抬头、税号、邮箱)→ 生成发票代码/号码/PDF;账单转「已开票」。默认沙箱,生产改 app.einvoice.provider 接航信/百望。

7.7 合同签约(自建电子签)

选模板(占位符自动填充)或手填 → 新建草稿 → 「发起签署」(内容定稿快照 + SHA-256 存证)→ 各方逐一签署 → 全部签署后合同生效并存证归档,金额>0 自动出账。可作废未签署合同;导出 CSV。

7.8 须知与授权

「须知管理」新建草稿(条款/隐私/公告)→ 发布(内容定稿、旧版归档);可设「强制确认」。「授权记录」查看同意/撤回(主体/版本/渠道/IP/时间),导出 CSV 取证。公开页 /pub/notices/active、/pub/consents。

7.9 智能客服

控制台右下角悬浮窗对话;基于产品知识库(FAQ)作答,支持流式与「转人工」;通用不绑厂商,未接大模型时降级为知识库匹配。本页可审计全部会话。

7.10 在线代码加固

  • 与「构建/打包」加固并存;「加固设置」按租户选模式(仅构建自建 / 仅平台在线 / 两者并存)+ 默认技术。
  • 「提交在线加固」上传可执行 jar,勾选技术(ProGuard 混淆 / 类加密+License 绑定 / 口令式类加密)→ 异步处理 → 下载产物。
  • 运行产物:java -Dharden.license=<.lic> -jar x.jar(License 绑定)/ java -Dharden.key=<口令> -jar x.jar(口令式)。

7.11 租户自助门户(面向最终客户)

运营在「自助门户」为租户「开通」生成访问码并复制凭证;客户访问门户地址 /portal,用「租户编号 + 访问码」登录,只读查看本租户 License/合同/账单/订阅(严格隔离)。

8. 客户端 SDK 接入与代码保护

  • SDK 验签:sdk/license-sdk 用公钥验 .lic 签名 + 功能/版本门禁(SemVer)。私钥不出域。
  • 构建时代码保护:examples/protected-app,mvn -Pharden 做 ProGuard 混淆;BuildProtected 加密关键类、解密密钥与 License 绑定(demo.sh 演示合法/错钥/篡改)。
  • 在线加固:见 7.10(平台上传加固,无需自建流水线)。
  • 设计详见 docs/02-代码保护方案.md、docs/03-License认证方案.md。

9. 智能客服大模型接入

通用 OpenAI 兼容,换厂商只改 .env 三项:

# 智谱 GLM-4-Flash(常免费)
AI_ENABLED=true AI_BASE_URL=https://open.bigmodel.cn/api/paas/v4 AI_API_KEY=*** AI_MODEL=glm-4-flash
# 本地 Ollama(离线)
AI_ENABLED=true AI_BASE_URL=http://ollama:11434/v1 AI_MODEL=qwen2.5:7b-instruct

选型/部署/成本对比详见 docs/智能客服-大模型选型与部署.md。

10. 运维

  • 备份:PostgreSQL/MySQL 定期 pg_dump/mysqldump + PITR;Redis 哨兵/集群(可选);加固产物目录 app.harden.work-dir 用持久卷。
  • 监控:Actuator + Prometheus(/actuator/prometheus)+ Grafana(预置看板 infra/grafana)。
  • 密钥:私钥与 JWT_SECRET、各渠道密钥经 env/Nacos/KMS 注入,不落明文(见 docs/密钥管理-KMS.md)。
  • 多实例:APP_REDIS_ENABLED=true 时心跳 nonce 走 Redis,可水平扩展;签发与在线校验可分离部署。
  • 升级:改 Flyway 迁移(同时补 db/mysql 方言)→ bash deploy/sql/generate-schema.sh 重生成 schema → CI 真机矩阵验证(PG/MySQL 5.7·8.0)。
