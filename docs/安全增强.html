安全增强：密钥轮换 / SDK 通道 mTLS / 限流防刷

1. 密钥轮换（多 kid / JWKS）——已落地

  • 每张 License 的声明含 sigAlg(算法) 与 kid(密钥标识,由公钥 SHA-256 派生)。
  • 平台暴露 JWKS 风格公钥集：GET /api/licenses/public-keys → [{kid, algorithm, publicKeyBase64}]。
  • SDK 按 kid 选公钥验签：LicenseVerifier.verify(lic, Map<kid, pubBase64>)(见 KidRotationTest)。

轮换流程（无停机）：

  1. 生成新密钥对(新 kid),与旧密钥并存;/api/licenses/public-keys 同时发布新旧公钥。
  2. 签发服务切到新私钥(新签 License 带新 kid);旧 License 仍可用旧公钥验签。
  3. 待旧 License 全部过期/续期后,从 JWKS 下线旧公钥。

当前实现：单活跃密钥 + kid + JWKS 端点 + SDK 多公钥验签能力(轮换基础设施齐备)。 生产：私钥托管 KMS/HSM,按上面流程并存多 kid;客户端定期刷新 JWKS 缓存。

2. SDK 通道限流/防刷——已落地

  • SdkRateLimitFilter 对 /sdk/** 按客户端 IP 每分钟固定窗口计数,超限返回 429 + Retry-After。
  • 配置：app.sdk.rate-limit-per-min(默认 120,环境变量 APP_SDK_RATE_LIMIT)。
  • 多实例生产应换 Redis 计数(INCR + EXPIRE),与 nonce 同一 Redis。
  • 实测：130 次/分钟突发心跳触发 10 次 429。

3. SDK 通道 mTLS——配置指引（部署期启用）

SDK 在线通道(/sdk/activate、/sdk/heartbeat)建议启用双向 TLS,确保只有持客户端证书的合法客户端可接入。

方式 A：在网关/nginx 终止 mTLS（推荐）

server {
    listen 443 ssl;
    ssl_certificate     /etc/nginx/cert/server.crt;
    ssl_certificate_key /etc/nginx/cert/server.key;
    ssl_client_certificate /etc/nginx/cert/client-ca.crt;  # 签发客户端证书的 CA
    ssl_verify_client on;                                   # 强制校验客户端证书
    location /sdk/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header X-SSL-Client-Verify $ssl_client_verify;
        proxy_set_header X-SSL-Client-DN     $ssl_client_s_dn;
    }
}

方式 B：Spring Boot 直接启用 mTLS（application-mtls.yml，按需引入）

server:
  ssl:
    enabled: true
    key-store: classpath:server-keystore.p12
    key-store-password: ${SSL_KEYSTORE_PWD}
    key-store-type: PKCS12
    client-auth: need               # 强制客户端证书
    trust-store: classpath:client-truststore.p12
    trust-store-password: ${SSL_TRUSTSTORE_PWD}

仅对 SDK 通道强制 mTLS、运营后台/前端用普通 HTTPS 时,推荐方式 A 在网关分路径配置; 证书签发与吊销纳入 PKI 流程,客户端证书可与 License/租户绑定以便溯源。
