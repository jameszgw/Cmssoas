# 功能说明：在线代码加固(上传 jar → 自动加固 → 下载)

在运营平台内提供「上传 jar → 在线自动加固 → 下载产物」的能力。**与「构建/打包时加固」并存、不取代**;
由运营用户在后台设置并可切换模式(按租户默认 + 任务可覆盖)。沿用平台一贯设计:RBAC、审计、异步任务、可插拔 provider。

## 两种加固方式(并存)
| 方式 | 何时 | 说明 |
|---|---|---|
| **构建/打包时** | 厂商自己的打包流水线 | ProGuard 混淆 + 关键类加密(解密密钥与 License 绑定),见 `examples/protected-app`(`mvn -Pharden`) |
| **平台在线** | 运营平台上传 jar | 本功能:上传可执行 jar,平台异步加固后下载 |

## 模式与切换(运营用户设置)
- **按租户**配置「加固模式」:`BUILD`(仅构建自建) / `ONLINE`(仅平台在线) / `BOTH`(并存)。
- **任务可覆盖**:每次提交可单独选择技术、绑定 License、口令、加密包前缀。
- 当某租户模式为 `BUILD` 时,**拒绝**其在线加固请求(提示改为 ONLINE/BOTH)。留空租户=平台级。

## 在线加固技术(可叠加,按勾选顺序执行)
1. **OBFUSCATE**:ProGuard 混淆(库内调用;保守默认:不裁剪/不优化、保留 `main` 与注解,内部成员改名)。可后续接入商用器。
2. **ENCRYPT_BIND**:关键类 AES-256-GCM 加密,密钥由**绑定的 License 的 .lic 原文**派生 → 运行期必须提供该 .lic 才能解密运行(与授权强绑定)。
3. **FATJAR_ENCRYPT**:关键类 AES 加密,密钥由**口令**派生 → 运行期提供口令解密。

加密产物结构:核心类被替换为 `enc/<path>.class.enc`(明文移除),注入纯 JDK 运行期 `HardenLauncher`/`HardenClassLoader`,
清单 `Main-Class` 改为启动器并记录 `Harden-Real-Main`/`Harden-Mode`。**未匹配前缀的依赖类保持明文正常加载**。

## 运行加固后的产物
```bash
# 与 License 绑定(ENCRYPT_BIND)
java -Dharden.license=/path/to/customer.lic -jar hardened-app.jar
# 口令式(FATJAR_ENCRYPT)
java -Dharden.key=<口令> -jar hardened-app.jar   # 或环境变量 HARDEN_KEY
```
缺少 .lic/口令或不匹配 → 启动器拒绝并解密失败(类无法加载)。

## 可插拔(接商用器)
`HardenProvider` 抽象(`technique()/available()/apply()`),内置 ProGuard 与 AES 两类实现;
接 Allatori / ClassFinal / Xjar 等只需新增实现并注册为 Bean,业务流水线无感。

## 接口
| 方法 | 路径 | 权限 | 说明 |
|---|---|---|---|
| GET | `/api/harden/config?tenantCode=` | harden:view | 取租户加固设置 |
| PUT | `/api/harden/config` | harden:config | 保存模式/默认技术 |
| GET | `/api/harden/jobs` / `/jobs/{id}` | harden:view | 任务列表 / 详情 |
| POST | `/api/harden/jobs` (multipart) | harden:run | 上传 jar 提交:`file` + `techniques`(CSV) + `tenantCode/bindLicense/passphrase/encryptPrefix` |
| GET | `/api/harden/jobs/{id}/download` | harden:view | 下载加固产物 |

## 配置
```yaml
app:
  harden:
    work-dir: ${HARDEN_WORK_DIR:./var/harden}   # 上传源 jar 与产物存放(生产建议挂载持久卷)
    max-file-mb: ${HARDEN_MAX_FILE_MB:200}
spring:
  servlet:
    multipart: { max-file-size: 200MB, max-request-size: 210MB }
```

## 迁移与权限
- `V18__harden.sql`:`harden_config` / `harden_job` + 权限点 `harden` / `harden:view` / `harden:run` / `harden:config`(随启动授予 SUPER_ADMIN)。
- MySQL 方言:`db/mysql/V18__harden.sql`。

## 验证
- `HardenIntegrationTest`(CI):编译微型 jar → FATJAR_ENCRYPT → 校验产物含 `enc/` 密文与注入启动器、清单入口改写。
- 真机已验证三模式可运行:OBFUSCATE、OBFUSCATE+FATJAR_ENCRYPT(`-Dharden.key`)、ENCRYPT_BIND(`-Dharden.license`)均能正确运行原程序;缺/错密钥则拒绝。
- 实现说明:ProGuard 经 `log4j2.component.properties` 强制 log4j-core 工厂,避免与 Spring 的 log4j-to-slf4j 冲突。
