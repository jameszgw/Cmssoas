## 0. 目标与范围

| 维度 | 目标 |
|------|------|
| 防逆向 | 通过纵深防御（混淆 + 字节码加密 + 关键逻辑 Native 化 + 反调试/反 dump）显著抬高逆向、脱壳、破解的成本与门槛。 |
| License 认证 | 支持**离线**（文件授权 + 硬件指纹绑定）与**在线**（心跳校验 + 实时吊销 + 浮动授权）两种模式；License 可管理、可续期、可变更、可吊销，并具备**版本号**与**历史追溯**。 |
| 多租户平台 | 提供租户开通、DB 与初始数据自动化初始化、初始超管账号下发；以"产品—模块—功能—版本"为核心的可运营授权模型，前后端齐全。 |
| 交付物 | 含**后端**（License Server / 运营后台服务）、**前端**（运营管理控制台）、**客户端 SDK**（嵌入被保护应用）三部分；每部分含设计思路、技术选型对比、潜在风险与应对。 |

> ⚠️ **诚实前提**：Java 字节码本质可逆，世界上不存在"绝对无法破解"的纯软件保护方案。本方案的目标是**抬高破解成本到超过软件价值**，并通过 License + 在线吊销 + 法律手段（LICENSE/合同）构建综合防线。所有"防护"均以"提高门槛 + 可发现 + 可追责"为现实目标，文中不会承诺"无法被破解"。

## 1. 总体架构

系统由三大子系统构成：

```
┌──────────────────────────────────────────────────────────────────┐
│                     运营方（厂商）侧                                  │
│                                                                    │
│  ┌────────────────────┐        ┌──────────────────────────────┐   │
│  │  运营管理控制台       │  HTTPS │   License Server / 运营后端    │   │
│  │  (前端 Vue3)        │◀──────▶│   (Spring Boot)              │   │
│  │  租户/授权/License   │        │                              │   │
│  │  历史追溯/统计       │        │  ┌────────┐  ┌────────────┐  │   │
│  └────────────────────┘        │  │签发引擎 │  │在线校验/吊销 │  │   │
│                                │  └───┬────┘  └─────┬──────┘  │   │
│                                │      │              │         │   │
│                                │  ┌───▼──────────────▼──────┐  │   │
│                                │  │   KMS / HSM（私钥）       │  │   │
│                                │  └─────────────────────────┘  │   │
│                                └──────────────┬───────────────┘   │
│                                               │ 运营 DB(多租户)     │
└───────────────────────────────────────────────┼──────────────────┘
                                                │ ① 离线: 下发 .lic 文件
                                                │ ② 在线: 心跳/校验 API
                                                ▼
┌──────────────────────────────────────────────────────────────────┐
│                      客户（被授权方）侧                              │
│   ┌───────────────────────────────────────────────────────────┐  │
│   │   被保护的 Spring Boot 应用（加密壳 + 混淆 + Native 校验）      │  │
│   │   ┌──────────────────────────────────────────────────┐    │  │
│   │   │  License SDK：指纹采集 / 验签 / 在线心跳 / 功能门禁  │    │  │
│   │   └──────────────────────────────────────────────────┘    │  │
│   └───────────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────────────┘
```

### 1.1 组件职责

| 组件 | 技术栈 | 职责 |
|------|--------|------|
| 运营管理控制台（前端） | Vue3 + TypeScript + Vite + Element Plus / Ant Design Vue + Pinia | 租户开通、套餐/授权配置、License 签发与生命周期管理、历史版本对比、用量与统计大盘、审计查询 |
| License Server / 运营后端 | Spring Boot 3.x + Spring Security + MyBatis-Plus/JPA + MySQL/PostgreSQL + Redis | 授权模型管理、License 签发（私钥签名）、在线校验/心跳、吊销列表（CRL）、用量计量、多租户数据隔离、审计日志 |
| KMS / HSM | 阿里云 KMS / 华为 KMS / 软 HSM（SoftHSM）/ 国密 SDF 设备 | 私钥安全存储与签名运算，私钥永不出域 |
| 客户端 License SDK | Java 库（jar），嵌入被保护应用 | 硬件指纹采集、公钥验签、离线 License 加载、在线心跳、功能/版本门禁、时间回拨检测 |
| 代码保护工具链 | 混淆器 + 字节码加密 + GraalVM/JNI + 构建插件 | 在 CI/CD 中对客户交付 jar 进行混淆、加密套壳、关键逻辑 Native 化 |

### 1.2 端到端核心流程（概览）

**离线授权流程**

1. 客户在目标机器运行"指纹采集工具"（SDK 提供），生成机器指纹 `machineCode`。
2. 客户将 `machineCode` + 订单号提交给运营方。
3. 运营人员在控制台选择租户/套餐/版本/有效期 → 后端用**私钥签名**生成 `.lic` 文件（绑定该指纹）。
4. 客户导入 `.lic` 到应用 → SDK 用内置**公钥验签** + 校验指纹/有效期/版本/功能 → 放行。

**在线授权流程**

1. 应用启动时携带 License 凭证（或激活码）向 License Server 注册/激活。
2. SDK 定期发送**心跳**（带 nonce 防重放、mTLS）→ 服务端校验有效性、并发席位、是否被吊销 → 返回**服务端签名**的最新授权快照。
3. 网络抖动时启用**宽限期（grace period）**离线容忍；超期或被吊销则降级/停服。

## 2. 设计原则

1. **纵深防御（Defense in Depth）**：任何单点都可能被攻破，故"混淆 + 加密 + Native + License + 在线吊销 + 审计"层层叠加，攻击者需要逐层突破。
2. **非对称信任根**：License 用厂商私钥签名、客户端用公钥验签。客户端**永不持有私钥**，因此即使被完全逆向，也无法**伪造**合法 License（只能尝试 patch 校验逻辑，而这正是代码保护层要拦截的）。
3. **失败策略分级**：
   - 验签失败 / 指纹不匹配 / 已过期 / 被吊销 → **拒绝启动或核心功能降级**（fail-closed）。
   - 在线校验网络异常 → 进入**宽限期**（fail-open 有限期），避免误伤生产。
4. **可观测、可追溯、可追责**：所有签发、变更、吊销、激活、心跳异常均落审计日志，支撑历史追溯与法务取证。
5. **运营友好**：授权模型与计费/套餐解耦但联动，支持自助开通、批量签发、到期预警、续费、灰度升级。
6. **密钥分级与轮换**：按产品线/大版本使用不同签名密钥；私钥托管 KMS/HSM，支持轮换与吊销。

## 3. 技术栈总表

| 层 | 选型 | 备选 | 说明 |
|----|------|------|------|
| 后端框架 | Spring Boot 3.x (Java 17/21) | Quarkus | 与客户交付物同栈，团队熟悉 |
| 安全 | Spring Security + JWT(运营后台) + mTLS(SDK 通道) | Sa-Token | 控制台 RBAC + SDK 双向认证 |
| 持久层 | MyBatis-Plus | Spring Data JPA | 多租户拦截器易实现 |
| 数据库 | PostgreSQL（推荐）/ MySQL 8 | TiDB | PG 的 RLS 行级安全利于多租户 |
| 缓存/限流 | Redis | —— | 心跳计数、并发席位、CRL 缓存 |
| 数据库迁移 | Flyway | Liquibase | 租户初始化与版本化 schema |
| 密钥 | 云 KMS / SoftHSM / 国密 SDF | TrueLicense KeyStore | 私钥不出域 |
| 签名算法 | Ed25519（推荐）/ ECDSA P-256 / SM2(国密) | RSA-2048 | 短签名、高性能 |
| 前端 | Vue3 + TS + Vite + Pinia + Element Plus / Ant Design Vue | React + Antd | 中后台生态成熟 |
| 图表 | ECharts | —— | 用量/统计大盘 |
| 混淆 | Allatori（商用）/ ProGuard（免费） | yGuard、DashO | 名称/控制流/字符串加密 |
| 字节码加密 | Xjar / ClassFinal | 自研 ClassLoader+Agent | 套壳 + 运行期解密 |
| Native 化 | GraalVM native-image / JNI(C/C++) | —— | 关键校验逻辑 AOT/原生化 |
| 打包为可执行 | jpackage / launch4j / exe4j | —— | 可选，分发便利（非核心防护） |
| 客户端 SDK | 纯 Java 库 + 可选 native agent | —— | 指纹/验签/心跳/门禁 |

## 5. 关键风险与应对（总览，详见各分册）

| 风险类别 | 风险 | 应对 |
|----------|------|------|
| 防逆向 | 字节码可被反编译；自定义 ClassLoader 解密的类可被内存 dump | 混淆+加密+Native 纵深；JVMTI 反 dump agent；关键校验下沉 Native；分散校验点 |
| License | 伪造 License | 非对称签名，私钥不出域，客户端仅验签 |
| License | patch 掉验签逻辑（NOP 校验） | 校验逻辑 Native 化、完整性自校验、多点冗余校验、在线二次校验 |
| License | 一份授权多机共用 | 硬件指纹绑定 + 在线并发席位控制 |
| License | 篡改系统时间绕过有效期 | 可信时间源 + 单调用量记录 + 时间回拨检测 |
| 在线 | 网络中断导致生产停摆 | 宽限期 fail-open；离线 License 兜底 |
| 密钥 | 私钥泄露导致全线失守 | KMS/HSM 托管、密钥分级、可轮换/可吊销 |
| 多租户 | 租户间数据越权 | 隔离策略 + 全局租户上下文拦截 + 行级安全 |
| 运营 | 到期未续导致客户业务中断 | 到期预警、续期宽限、平滑降级 |

> 详细的"潜在风险 / 应对措施"在每个分册的对应章节末尾给出。
