# 05 · 安全密钥管理与部署架构

> 整个方案的"信任根"是签名私钥。私钥一旦泄露，所有 License 均可被伪造，全线失守。因此密钥管理是重中之重。

---

## 1. 密钥体系

| 密钥 | 用途 | 存放 | 暴露面 |
|------|------|------|--------|
| **License 签名私钥** | 签发 License | KMS/HSM，永不出域 | 仅 License Server 通过 KMS API 调用签名 |
| License 验签公钥 | 客户端验签 | 内置于 SDK/被保护应用 | 可公开（公钥泄露无害） |
| 产品根密钥（解密密钥派生） | 派生 class 解密密钥（02 §4） | KMS + 构建机隔离 | 不随交付物分发 |
| mTLS 证书 | SDK↔Server 通道 | 双方证书库 | 受 CA 管控 |
| 后台 JWT 签名密钥 | 运营后台会话 | 服务端配置中心/KMS | 服务端 |

### 1.1 密钥分级与轮换
- **按产品线 + 大版本**使用不同签名密钥（`kid` 标识），缩小爆炸半径。
- 客户端内置**多公钥**（按 `kid` 选择），支持平滑**轮换**：新 License 用新私钥签，旧 License 用旧公钥继续验。
- 私钥泄露应急：吊销对应 `kid` → 用新密钥重签存量 License → 推送更新（在线模式可远程刷新）。

### 1.2 选型对比

| 方案 | 安全性 | 成本 | 信创合规 | 适用 |
|------|--------|------|----------|------|
| 云 KMS（阿里/华为/腾讯） | 高（私钥不出域、审计、轮换） | 中（按调用计费） | 部分支持国密 | 公有云/混合云 |
| 硬件 HSM（加密机） | 最高（物理防护、抗提取） | 高（采购+运维） | 支持国密 SDF | 政企/金融/私有化 |
| SoftHSM / 软实现 | 中（依赖主机安全） | 低 | —— | 开发/测试 |
| TrueLicense KeyStore | 低-中（密钥随包易被提取） | 低 | —— | 不推荐用于高价值场景 |

> **结论**：生产用**云 KMS** 或**硬件 HSM**；金融/政企信创选支持**国密 SM2** 的 HSM/加密机。开发测试可用 SoftHSM。

## 2. 通道与接口安全

- **SDK ↔ Server**：mTLS 双向认证 + 请求签名（HMAC + nonce + 时间戳）防中间人/重放；响应由服务端签名，SDK 验签。
- **控制台 ↔ 后端**：HTTPS + JWT + RBAC + 关键操作 MFA/审批 + 防重放 + 接口限流。
- **最小权限**：签发权限、吊销权限分离；私钥调用仅签发引擎服务账户可用。
- **审计**：所有签发/吊销/密钥调用入不可变审计日志，支持告警（异常签发量）。

## 3. 部署架构

```
                         ┌──────────────── WAF / API 网关 ────────────────┐
                         │           (TLS 终止、限流、防重放)               │
                         └───────┬───────────────────────────┬────────────┘
                                 │                           │
                    ┌────────────▼─────────┐     ┌───────────▼───────────┐
                    │ 运营后端(签发/管理)    │     │ 在线校验服务(心跳/CRL)  │
                    │ 多实例, 内网          │     │ 多实例, 独立扩缩容       │
                    └─────┬──────────┬─────┘     └───────────┬───────────┘
                          │          │                       │
                ┌─────────▼──┐  ┌────▼─────┐         ┌───────▼────────┐
                │ KMS / HSM  │  │ PostgreSQL│         │     Redis      │
                │ (私钥)     │  │ (主从+备份)│         │ (席位/nonce/CRL)│
                └────────────┘  └──────────┘         └────────────────┘
```

- **签发服务**与**在线校验服务**分离：前者低频高安全（靠近 KMS、严格管控），后者高并发（独立扩缩容）。
- **高可用**：服务无状态多实例；DB 主从 + 定期备份 + 异地容灾；Redis 哨兵/集群；KMS/HSM 双机。
- **私有化交付**：提供单机/内网部署包（含离线签发能力、SoftHSM/本地加密机适配），满足无外网客户。
- **可观测**：Prometheus + Grafana 监控签发量、心跳 QPS、宽限期触发、异常告警；日志集中（ELK）。

## 4. 潜在风险与应对

| 风险 | 应对 |
|------|------|
| 私钥泄露 | KMS/HSM 不出域 + 分级 + 轮换 + 应急重签 + 异常签发告警 |
| KMS/HSM 不可用致无法签发 | 双机热备 + 限流排队 + 离线应急签发预案 |
| 在线服务被 DDoS | WAF + 网关限流 + 弹性扩容 + CDN/边缘 |
| DB 故障/数据丢失 | 主从 + PITR 备份 + 异地容灾 + 定期恢复演练 |
| 证书过期致 SDK 断连 | 证书到期监控 + 自动轮换 + 宽限期兜底 |
| 内部人员越权签发 | 权限分离 + 审批流 + 全审计 + 定期审计复核 |
| 私有化环境密钥保护弱 | 强制本地加密机/SoftHSM + 交付即轮换专属密钥 |
