06 · 实施路线图与风险总览

1. 分阶段实施路线图

阶段一：MVP（4–6 周）—— 跑通"签发→验证"主链路

  • 后端：License 签名/验签（Ed25519）、claims 模型、离线 .lic 签发、KMS 接入（先 SoftHSM）。
  • SDK：硬件指纹、公钥验签、离线校验、功能门禁注解、有效期/版本校验。
  • 平台：最简租户/套餐/License 签发界面 + 历史记录。
  • 代码保护：ProGuard 混淆 + Xjar/ClassFinal 套壳跑通 1 个示例应用。
  • 里程碑：一个被保护的 Spring Boot 应用，凭离线 License 在绑定机器上运行，换机失效。

阶段二：在线 + 运营化（4–6 周）

  • 在线激活/心跳/CRL/并发席位/宽限期；可信时间与回拨检测。
  • 平台：产品-模块-功能-版本矩阵、套餐编排、订阅、License 生命周期（续期/变更/吊销）、版本 diff、审计。
  • 多租户：租户开通 Saga、Flyway 初始化、超管下发、PG RLS。
  • 前端控制台：开通向导、License 管理、历史追溯、运营大盘。
  • 里程碑：可在控制台完成"开通租户→配置授权→签发→在线吊销生效"。

阶段三：加固与信创（4–6 周）

  • 关键逻辑 JNI Native 化 + OLLVM；反调试/反 dump/完整性自校验；解密密钥依赖 License。
  • 生产 KMS/HSM，国密 SM2/SM3 适配；mTLS。
  • GraalVM native-image 试点核心模块。
  • 高可用、监控告警、私有化离线交付包。
  • 里程碑：通过内部红队渗透测试，破解成本评估达标。

阶段四：运营完善（持续）

  • 计费/订单对接、到期预警自动化、批量签发、自助续费、水印溯源、数据大盘完善。

4. 现实边界声明（重要）

  • 没有绝对安全：纯软件保护只能"抬高门槛 + 提高发现/追责能力"，无法 100% 阻止逆向。
  • 综合防线：技术（代码保护+License）+ 运营（在线吊销+水印溯源）+ 法律（授权协议+合同）三位一体。
  • 成本权衡：防护强度与开发/适配/性能成本正相关，应按软件价值与客户环境分级实施（如内网涉密重离线+强加固，SaaS 重在线+席位）。
  • 持续对抗：破解与防护是持续博弈，需保留升级能力（密钥轮换、防护策略热更新、版本迭代）。
